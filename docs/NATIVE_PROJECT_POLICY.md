# Native Project Policy

## Current Status: Generated (Not Committed)

The `/ios/` and `/android/` directories are currently **gitignored** and treated as generated outputs. They are regenerated locally via `bun run cap:sync`.

This is the correct policy for the current phase because:

- The app uses a **hosted-shell** architecture (WebView loads `https://morphic.sh`)
- No custom native platform config exists yet (no deep link entitlements, no push config, no custom splash/icons)
- Native projects are disposable ŌĆö any developer can regenerate them from `capacitor.config.ts`

## When to Commit Native Projects

Commit `/ios/` and `/android/` when **any** of the following becomes true:

- [ ] Deep links require Associated Domains (iOS) or intent filters (Android)
- [ ] Push notifications require APNs entitlements or FCM `google-services.json`
- [ ] Custom app icons and splash screens are designed and placed in native assets
- [ ] iOS `Info.plist` or Android `AndroidManifest.xml` has non-default permissions
- [ ] App Store / Play Store metadata needs to be version-controlled
- [ ] Native plugins require platform-specific configuration files

## How to Commit Native Projects

When the time comes:

```bash
# 1. Remove from .gitignore
sed -i '' '/^\/ios\//d' .gitignore
sed -i '' '/^\/android\//d' .gitignore

# 2. Generate fresh native projects
bun run cap:sync

# 3. Inspect generated files
# Remove any local signing/secrets:
rm -rf ios/App/Pods  # CocoaPods are generated
# Check for .env, keystore, provisioning profiles ŌĆö remove them

# 4. Stage and commit
git add ios/ android/
git commit -m "feat: commit native projects for platform config"

# 5. Add docs update
# Update this file to say "Status: Committed"
```

## What Must NOT Be Committed

| Item                                  | Reason                                            |
| ------------------------------------- | ------------------------------------------------- |
| `ios/App/Pods/`                       | Generated by CocoaPods ŌĆö add to .gitignore        |
| `*.xcworkspace/xcuserdata/`           | User-specific Xcode data                          |
| `*.keystore` / `*.jks`                | Android signing keys ŌĆö secrets                    |
| `*.mobileprovision`                   | iOS provisioning profiles ŌĆö secrets               |
| `google-services.json` with real keys | Should use CI env vars or a gitignored local copy |
| Any `.env` files inside native dirs   | Checked by `cap:verify`                           |

## CI Considerations

Once native projects are committed:

1. **Android debug build** ŌĆö add CI job with `./gradlew assembleDebug`
2. **iOS project validation** ŌĆö add CI job (requires macOS runner) with `xcodebuild -project ... -scheme ... build`
3. **Capacitor sync verification** ŌĆö ensure `cap sync` produces no diff after a fresh checkout
